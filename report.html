Project 2 Report - Milestone 2

Requirements Checklist

  1. Git Repository: This project uses the git repository created by the professor.
    Repository URL: https://github.com/REPLACE-WITH-YOUR-REPO
    Note: Incremental commits have been made throughout the development process.
  2. Published Site: The site is published to the w3.cs.jmu.edu server.
    Deployed URL: https://w3stu.cs.jmu.edu/kimbalbx/cs343/project/
  3. Responsive Design with Media Queries: The site supports distinct layouts for large and small screens.
    See styles/style.css starting at line 847 where media queries are implemented with comments.
    Key responsive features:
    • Navigation switches from horizontal to wrapped/vertical layout on small screens
    • Pet container changes from row (desktop) to column (mobile) layout
    • Customization controls stack vertically on mobile devices
    • Activity buttons use CSS Grid that adapts from multi-column to single column
  4. Five Separate Pages: The site contains six pages (five required + report page), each with meaningful purpose:
    • index.html - Home page displaying the pet and weather status
    • play.html - Interactive play activities with the pet
    • customize.html - Pet customization interface
    • about.html - Information about the project and developer
    • contact.html - Resources and contact form
    • report.html - This project report (additional page)
    All pages are logically linked through the navigation menu.
  5. Header: Every page includes a consistent header with the PixelPaws logo and site title.
    The header is sticky and remains visible while scrolling.
  6. Navigation Links: A navigation menu is present in the header of every page.
    The active page is highlighted using the .active class.
  7. Images with Alt Attributes: All images include descriptive alt text:
    • Logo: "PixelPaws logo with a cute pixelated pet icon"
    • Pet preview: "Your customizable virtual pet - a cute gray cat"
    Alt text provides meaningful descriptions for screen readers and web crawlers.
  8. HTML and CSS Validation: All pages and stylesheets validate without errors.
    Validated using:
    • Nu HTML Checker
    • WAVE Accessibility Evaluator
  9. Multiple Font Faces: The site uses two different font families:
    • Georgia (serif): Used for all headings (h1-h6)
    • Arial (sans-serif): Used for body text and navigation
    See CSS variables --font-heading and --font-body in style.css.
  10. User Interaction: The site includes multiple forms for user interaction:
    • Customization form (customize.html): Text input for pet name, reset and save buttons
    • Contact form (contact.html): Name, email, subject, and message inputs with validation
    • Interactive buttons: Arrow buttons for customization, activity buttons for play
    Note: JavaScript functionality will be implemented in Milestone 3. Forms currently display properly with proper HTML structure and validation attributes.
  11. External Links: The site includes links to multiple external sites:
    • MDN Web Docs - Web development documentation
    • WCAG Quick Reference - Accessibility guidelines
    • OpenWeatherMap API - Weather API (planned integration)
    • JMU Counseling Center - Student wellness resource
    • Additional resources on the Resources page
    All external links use target="_blank" and rel="noopener noreferrer" for security.

Professional Practices

  • Code Formatting: All HTML and CSS follows consistent formatting standards using VS Code's built-in formatter.
  • File Organization: Resources are properly organized into images/, styles/, scripts/, and design/ directories.
  • Aesthetics: The site uses a consistent purple color scheme based on the design mockups, with proper spacing, alignment, and visual hierarchy.
  • Coherent Style: Shared styles are in styles/style.css, maintaining consistency across all pages.
  • Content Alignment: Content uses left-alignment with clear axes. Flexbox and Grid layouts ensure proper spacing.
  • Readable Code: Uses semantic HTML5 elements, kebab-case naming conventions, and CSS custom properties for maintainability.
  • Accessibility: Implements ARIA labels, skip links, semantic markup, and follows WCAG Level A guidelines.
  • File Naming: All filenames are lowercase with only alphanumeric characters and hyphens.
  • Comments: Code includes meaningful comments explaining major sections and responsive breakpoints.

Additional Progress Notes

Design Implementation: The static site closely follows the design mockups provided in the design/ directory, including the desktop customization page and mobile home page layouts.

Pet Graphics: The pet is styled using CSS for a charming, customizable appearance. The design uses CSS shapes and positioning to create the cat character.

Weather Theming: Weather-based background styling is implemented (sunny, rainy, cloudy themes) ready for API integration in Milestone 3.

Accessibility Features:

  • Skip to main content link for keyboard navigation
  • ARIA labels on all interactive elements
  • Proper heading hierarchy
  • Focus indicators for keyboard users
  • High contrast mode support
  • Reduced motion preferences respected
